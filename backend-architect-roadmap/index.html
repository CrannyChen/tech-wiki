關聯式資料庫 SQL

PostgreSQL / MySQL / Oracle

+
  • 索引原理：B+Tree、複合索引、覆蓋索引、索引失效SQL 效能調優第一課
  • 交易 ACID & 隔離等級：Read Uncommitted → Serializable髒讀、幻讀、不可重複讀 解決方案
  • EXPLAIN / EXPLAIN ANALYZE 分析執行計劃找出慢查詢的工具
  • Schema 設計：正規化 1NF~3NF、反正規化效能 vs 一致性的取捨
  • 分區(Partition)、分表、讀寫分離TimescaleDB 超級表就是分區概念的實際應用
  • 連接池：HikariCP 配置調校Spring Boot 預設連接池
// 學習路徑
01.對手邊的 PostgreSQL 下複雜查詢，用 EXPLAIN ANALYZE 觀察執行計劃
02.刻意製造慢查詢（大量資料 + 無索引）→ 加索引後觀察效能差異
