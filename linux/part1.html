cat / zcat — 讀取檔案內容

基礎指令 log 分析常用
cat 是 Linux 最基本的讀檔工具；zcat 是它的壓縮版，可以直接讀 .gz 檔案，在看舊 log 時非常實用。

cat 完整用法

cat

cat = concatenate（串接），本來是用來把多個檔案合併輸出，但最常用來「看檔案內容」。

bash — 基本用法
# 1. 最基本：顯示檔案內容 $ cat /etc/hostname ubuntu-server # 2. -n 顯示行號（debug 很好用） $ cat -n server.xml 1 <?xml version="1.0" encoding="UTF-8"?> 2 <Server port="8005" shutdown="SHUTDOWN"> # 3. -A 顯示特殊字元（找換行問題） $ cat -A config.txt key=value$ ← $ 代表正常換行(\n) key2=val2^M$ ← ^M 代表 Windows 換行(\r\n)，會造成 Linux 問題！ # 4. 合併多個檔案 $ cat file1.txt file2.txt > combined.txt # 5. 建立簡單檔案（Ctrl+D 結束） $ cat > memo.txt 這是我打的內容 ^D（Ctrl+D 送出）

🔍 cat vs less vs tail — 怎麼選？

情況用什麼原因
檔案很小，想全看cat一次輸出全部
檔案很大，要上下捲動less分頁顯示，q 離開
看最後幾行 logtail -n 50只顯示最後 N 行
即時監看 logtail -f持續追蹤新增內容

zcat — 直接讀壓縮檔

zcat

Linux 的 log rotation 會自動把舊 log 壓縮成 .gz，zcat 可以直接讀取，不用先 gunzip 解壓。

ℹ️
Log Rotation 是什麼？ 系統每天會把 syslog → syslog.1 → syslog.2.gz → 自動刪除。 你要查 3 天前的 log 就需要 zcat 來讀 .gz 檔。
bash — zcat 用法
# 直接讀 .gz 檔 $ zcat /var/log/syslog.2.gz # 搭配 grep 篩選關鍵字（最實用） $ zcat /var/log/syslog.2.gz | grep "ERROR" # 搭配 less 分頁閱讀 $ zcat app.log.gz | less # 查所有壓縮 log 的 ERROR（*萬用字元） $ zcat /var/log/syslog.*.gz | grep "Failed" # 顯示行數（解壓後有多少行） $ zcat big.log.gz | wc -l 48320
💡
同族指令：zcat / zgrep / zless / zmore — 都是對應普通版本，但能直接吃 .gz

zip / unzip — Windows 通用格式

壓縮 跨平台
.zip 是 Windows / Mac / Linux 都支援的格式，跟別人交換檔案或上傳到 Windows 伺服器時很常用。

zip 壓縮

壓縮
bash
# 壓縮單一檔案 $ zip output.zip myfile.txt # -r 壓縮整個目錄（recursive，必要！） $ zip -r myapp.zip mydir/ # -9 最高壓縮率（慢），-1 最快（大） $ zip -r -9 myapp.zip mydir/ # -e 加密（會要求輸入密碼） $ zip -r -e secret.zip mydir/ # 排除特定檔案（不打包 .class 和 node_modules） $ zip -r myapp.zip mydir/ -x "*.class" -x "*/node_modules/*" # 查看 zip 內容（不解壓） $ zip -sf myapp.zip

unzip 解壓縮

解壓
bash
# 解壓到目前目錄 $ unzip myapp.zip # -d 指定目的目錄 $ unzip myapp.zip -d /opt/myapp # -l 列出內容（不解壓） $ unzip -l myapp.zip Length Date Time Name --------- ---------- ----- ---- 10240 2024-04-16 09:00 mydir/app.jar # -o 強制覆蓋（不問確認） $ unzip -o myapp.zip -d /opt/myapp # 解壓時中文亂碼？Windows zip 常見問題 $ unzip -O CP950 windows.zip
⚠️
中文亂碼：Windows 的 zip 用 CP950 編碼，在 Linux 解壓會亂碼，加 -O CP950 可解決

tar — Linux 標準打包工具

.tar.gz Linux 標準
tar 是 Linux 最常見的打包方式，搭配 gzip（-z）變成 .tar.gz，同時打包多個檔案並壓縮。

tar 參數完整解說

參數
參數全名說明
-cCreate建立新的壓縮檔
-xeXtract解壓縮
-tlisT列出壓縮檔內容（不解壓）
-zgZip透過 gzip 壓縮，產生 .gz
-jbzip2透過 bzip2 壓縮，更小但更慢
-vVerbose顯示處理過程（每個檔名）
-fFile指定檔名，必須放最後
-CChange dir指定解壓目錄
-pPreserve保留原始權限
--exclude—排除特定檔案
💡
記憶口訣：壓縮 = create zip verbose file → czvf；解壓把 c 換 x → xzvf

常用操作範例

指令
bash
## ─── 壓縮 ─────────────────────────────── # 打包目錄 $ tar -czvf myapp.tar.gz mydir/ # 打包多個目錄/檔案 $ tar -czvf backup.tar.gz /etc/nginx /var/log/nginx # 排除特定目錄 $ tar -czvf myapp.tar.gz mydir/ --exclude="mydir/node_modules" ## ─── 解壓縮 ────────────────────────────── # 解壓到當前目錄 $ tar -xzvf myapp.tar.gz # 解壓到指定目錄（目錄要先存在） $ tar -xzvf myapp.tar.gz -C /opt/deploy ## ─── 查看內容（不解壓）────────────────── $ tar -tzvf myapp.tar.gz drwxr-xr-x tomcat/tomcat 0 2024-04-16 09:00 mydir/ -rw-r--r-- tomcat/tomcat 52428800 2024-04-16 09:00 mydir/app.jar

grep / zgrep — 文字搜尋神器

必學指令 log 分析 正則表達式
grep 是 Linux 最重要的文字過濾工具，任何 log 分析、設定查找都離不開它。搭配 pipe 使用威力倍增。

grep 常用選項速查

指令
選項說明範例
-i忽略大小寫grep -i "error" app.log
-n顯示行號grep -n "Exception" app.log
-r遞迴搜尋目錄grep -r "TODO" src/

-c計算匹配行數grep -c "ERROR" app.log
-v反向（不包含）grep -v "DEBUG" app.log

-C N顯示前後各 N 行grep -C 3 "ERROR" app.log

實戰場景範例

實戰

## 場景1：查 Spring Boot log 的 ERROR $ grep -n "ERROR" catalina.out 1234:2024-04-16 09:12:33 ERROR c.e.MyService - NullPointerException ## 場景2：看 ERROR 前後 5 行（找上下文） $ grep -C 5 "NullPointerException" catalina.out ## 場景3：即時監看 log 同時過濾 $ tail -f catalina.out | grep --color "ERROR\|WARN" ## 場景4：排除不重要的 log $ cat catalina.out | grep -v "DEBUG" | grep -v "INFO" ## 場景5：統計今天有幾個 ERROR $ grep -c "2024-04-16.*ERROR" catalina.out 47 ## 場景6：搜尋設定檔中的 port 設定 $ grep -r "8080" /etc/nginx/ ## 場景7：找哪些檔案包含某個字串 $ grep -rl "datasource.url" /opt/myapp/config/ /opt/myapp/config/application.properties ## 場景8：多個關鍵字（OR） $ grep -E "ERROR|Exception|OOM" catalina.out

zgrep — 直接搜尋 .gz

zgrep

zgrep 的所有選項與 grep 完全相同，差別只是它可以讀 .gz 格式。

bash — zgrep

chmod / chown — 權限管理

權限 重要
Linux 的權限系統是安全的基礎，弄錯權限是很多「Permission denied」錯誤的根源。

rwx 數字對照

概念
r
Read 讀取
4
w
Write 寫入
2
x
Execute 執行
1
數字符號意義
7rwx讀+寫+執行（4+2+1）
6rw-讀+寫（4+2）
5r-x讀+執行（4+1）
4r--只讀（4）
0---無權限

三位數字依序代表：擁有者 / 群組 / 其他人

例如 755 = 擁有者 rwx，群組和其他人 r-x

chmod / chown 常用指令

指令
bash
## chmod — 改權限 # 數字方式（最常用） $ chmod 755 startup.sh # rwxr-xr-x $ chmod 644 config.xml # rw-r--r-- $ chmod 600 secret.key # rw------- (只有自己能讀寫) # 符號方式（+新增 -移除 =設定） $ chmod +x startup.sh # 所有人加執行權 $ chmod u+x script.sh # 只有擁有者加執行 $ chmod go-w file.txt # 群組和其他人移除寫入 # -R 遞迴整個目錄 $ chmod -R 755 /opt/tomcat ## chown — 改擁有者 # 改擁有者 $ sudo chown tomcat app.jar # 改擁有者和群組 $ sudo chown tomcat:tomcat app.jar # -R 遞迴 $ sudo chown -R tomcat:tomcat /opt/tomcat
💡
常見情境：Tomcat 的 bin/*.sh 需要 chmod +x 才能執行；WAR 檔需要 chown tomcat:tomcat Tomcat 才能讀取。

pipe 管線 & 重導向

進階技巧 組合威力
pipe（|）是把多個指令串接起來的機制，重導向（> >>）是控制輸出位置。兩者組合可以完成複雜任務。

pipe | 管線

pipe
概念：把前一個指令的「輸出」，當成後一個指令的「輸入」
cat app.log
讀出所有內容
|
grep "ERROR"
只留 ERROR 的行
|
wc -l
計算有幾行
bash — pipe 實例
# 今天的 ERROR 有幾筆 $ cat catalina.out | grep "ERROR" | wc -l 47 # 找最大的 10 個檔案 $ ls -lh | sort -k5 -rh | head -10 # 即時監看 log 並過濾 $ tail -f catalina.out | grep --color "ERROR" # 看哪個 process 用最多 CPU $ ps aux | sort -k3 -rn | head -5

重導向 > >> 2>

redirect
符號說明範例
>輸出到檔案（覆蓋）echo "hello" > file.txt
>>輸出到檔案（追加）echo "hello" >> file.txt
2>錯誤輸出到檔案command 2> error.log
2>&1錯誤和正常輸出合併command > all.log 2>&1
<從檔案讀取輸入mysql db < dump.sql
# 把 log 中的 ERROR 存到另一個檔 $ grep "ERROR" catalina.out > errors.txt # 追加（不覆蓋）到 log $ echo "部署完成 $(date)" >> deploy.log # 標準輸出和錯誤都存起來 $ java -jar app.jar > app.log 2>&1
